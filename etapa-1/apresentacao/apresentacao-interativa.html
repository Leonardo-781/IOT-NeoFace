O que é o Projeto?

Um sistema de monitoramento IoT distribuído que:

  • ✅ Coleta dados de múltiplos sensores ESP32 distribuídos
  • ✅ Transmite via MQTT para um servidor central
  • ✅ Processa e persiste em PostgreSQL/TimescaleDB
  • ✅ Visualiza em tempo real em Dashboard Web
  • ✅ Permite controle remoto de devices

🎯 Objetivo Principal

Demonstrar conceitos reais de Sistemas Distribuídos: comunicação entre processos, sincronização, tolerância a falhas e escalabilidade.

Por que é Distribuído?

🌍 Múltiplas Máquinas

Backend (Notebook) e BD (Raspberry Pi 3) em hosts diferentes

📡 Comunicação em Rede

MQTT TCP, HTTP, PostgreSQL via rede (mesmo em VPN com Tailscale)

🔌 Múltiplos Sensores

Vários ESP32 nodes publicando dados em paralelo

🔄 Sincronização

Necessidade de manter estado consistente entre componentes

Arquitetura Geral

ESP32 Nodes ──MQTT──→ Mosquitto Broker ──→ Backend (Node.js) ↓ PostgreSQL (Raspberry Pi 3) ↓ Dashboard Web (Acesso remoto)

Componentes Principais:

  • 7 ESP32 Nodes: Sensores (BMP280, MQ-7)
  • MQTT Broker: Mosquitto
  • API Gateway: Node.js :3001
  • Backend: Node.js :3000
  • BD: PostgreSQL/TimescaleDB

Conceitos de Sistemas Distribuídos

  • 📌 Processos/Threads: Node.js event-loop + cluster
  • 🔌 Sockets: MQTT (TCP), HTTP (TCP)
  • 📞 RPC: HTTP REST (futuro: gRPC)
  • 📨 Message Queue: MQTT Broker
  • 🔍 Naming Service: DNS + Tailscale MagicDNS
  • 🔒 Sincronização: BD shared state + eventual consistency
  • 🔐 Exclusão Mútua: PostgreSQL locks, transações ACID
  • 📊 Replicação: Futura com PostgreSQL replication
  • ❤️ Tolerância a Falhas: Health-checks, auto-restart
  • 🛡️ Segurança: RBAC, sessões, Tailscale VPN

Componentes Principais

🖥️ Backend (Node.js :3000)

Processa telemetria, gerencia BD, autenticação, UI web. Máquina: Notebook

💾 PostgreSQL (Raspberry Pi 3)

Persistência de série temporal (TimescaleDB), usuarios, alertas. IP: 192.168.1.10

🌐 API Gateway (Node.js :3001)

Recebe telemetria do gateway IoT, valida e repassa ao backend. Tunelado via Tailscale.

📡 Mosquitto MQTT

Broker que coordena pub/sub entre ESP32 nodes e backend. Porta 1883.

Fluxo de Dados

Telemetria (Entrada)

ESP32 Node ─MQTT Publish→ Mosquitto ─Subscribe→ Backend ↓ Processa ↓ PostgreSQL INSERT

Comando (Saída)

Dashboard ─POST /command→ Backend ─MQTT Publish→ Mosquitto ↓ ESP32 recebe
