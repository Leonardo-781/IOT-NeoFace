O Micro-mundo

Sistema IoT Distribuído para Monitoramento Ambiental em Tempo Real

Características

  • Coleta contínua de dados
  • Múltiplos nós geograficamente distribuídos
  • Sincronização centralizada
  • Dashboard web
  • Displays locais OLED

Sensores

  • Temperatura
  • Umidade
  • Pressão
  • Força de sinal WiFi (RSSI)
  • Qualidade de conexão

O que o Sistema Faz

  1. Coleta contínua de leituras de sensores (temperatura, umidade, pressão)
  2. Monitoramento de força e qualidade de sinal WiFi para diagnóstico
  3. Exibição em tempo real em telas OLED com animações responsivas
  4. Controle interativo via sensor de toque capacitivo
  5. Sincronização com servidor central através de API HTTP
  6. Persistência de séries temporais em PostgreSQL
  7. Dashboard web para visualização e controle remoto

Infraestrutura Física

Raspberry Pi 3B

  • PostgreSQL
  • Banco de dados persistente
  • IP fixo: 192.168.1.10
  • Conectividade: Ethernet

Notebook (Linux Mint)

  • Node.js API
  • Nginx proxy reverso
  • IP fixo: 192.168.1.11
  • Conectividade: WiFi

ESP32/ESP8266 (Distribuídos)

  • Múltiplos nós em diferentes locais
  • Coleta de dados via sensores
  • WiFi para comunicação
  • Displays OLED locais

Fluxo de Comunicação

ESP32/ESP8266 → [WiFi] → Nginx → API Node.js → PostgreSQL

Protocolo de Dados

HTTP POST com payload JSON

  • Frequência: 30-60s
  • Buffer local: 10 leituras
  • Auto-recover WiFi
  • Backoff exponencial
  • Rate limiting
  • Compressão gzip
  • Logging de requisições
  • CORS habilitado

Sensores Instalados

BMP280

  • Temperatura: -40 a +85°C
  • Pressão: 300-1100 hPa
  • Protocolo: I2C/SPI
  • Precisão: ±1°C

DHT11

  • Temperatura: 0-50°C
  • Umidade: 20-80%
  • Protocolo: Single-wire
  • Precisão: ±2°C, ±5%

RSSI (Received Signal Strength Indicator)

Monitoramento de qualidade de sinal WiFi:

  • Diagnóstico real de cobertura
  • Detecção de interferência
  • Planejamento de repetidores

Displays e Interatividade

Display OLED SSD1306

  • 128×64 pixels
  • Protocolo I2C
  • Taxa: 1-2 Hz
  • Conteúdo:
    • Temperatura em tempo real
    • Força de sinal WiFi
    • Animações de rosto
    • Status do sistema

Sensor Toque TTP223

  • Controle capacitivo
  • 5 expressões
  • Debounce 300ms
  • Ciclagem ao tocar

Schema PostgreSQL

Tabela: telemetry (TimescaleDB)

  • time (TIMESTAMP com timezone)
  • device_id (chave de particionamento)
  • temperature_c, humidity_percent, pressure_hpa
  • rssi_dbm, battery_voltage
  • Retenção: 90 dias

Tabela: devices

  • ID, nome, localização, status
  • Last seen, firmware version
  • IP address (último visto)

Tabela: alerts

Tipo, severidade, timestamp criação/resolução

Payload JSON - Exemplo

{ "device_id": "ESP32_SALA_001", "timestamp": 1715179200, "sensor_data": { "temperature_c": 24.5, "humidity_percent": 65.2, "pressure_hpa": 1013.5, "rssi_dbm": -45, "battery_voltage": 3.3 }, "display_state": { "current_expression": "happy", "screen_active": true }, "system_health": { "wifi_reconnects": 2, "api_failures": 0 } }

Objetivos de Sistemas Distribuídos

1. Escalabilidade

Novos nós podem ser adicionados sem redesenho. Nginx permite escalabilidade horizontal.

2. Disponibilidade

Buffer local em ESP, Nginx e API redundantes, PostgreSQL com snapshots automáticos.

3. Tolerância a Falhas

Reconexão automática, retry com backoff, health check, fila de mensagens locais.

4. Compartilhamento de Recursos

Nginx como ponto centralizado, PostgreSQL como estado único, transações ACID.

Desafios Reais do Sistema

Infraestrutura

  • IP dinâmico do provedor
  • Falhas WiFi frequentes
  • Latência variável
  • Nós geograficamente dispersos

Soluções

  • DHCP estático no roteador
  • Buffer local nos nós
  • Backoff exponencial
  • Sincronização periódica
